### 方向1：SQL能力（核心考点 · 必问）

这个岗位最核心的要求就是 **SQL取数能力**，面试大概率会有SQL题或场景题。

#### 预测问题1：写一个SQL，统计每个城市每个品牌的商机数量，并取前3名

**回答思路：**

```sql
SELECT 
    city,
    brand,
    COUNT(DISTINCT opportunity_id) AS opp_cnt
FROM opportunity_table
WHERE dt = '2026-05-29'  -- 取最新分区
GROUP BY city, brand
QUALIFY ROW_NUMBER() OVER (PARTITION BY city ORDER BY COUNT(DISTINCT opportunity_id) DESC) <= 3
ORDER BY city, opp_cnt DESC;
```

**讲解要点：**
- 强调用了 `QUALIFY` 或子查询 + `ROW_NUMBER()` 实现分组TopN
- 提到分区过滤 `dt=` 是数仓查询的常规优化
- 展示你对窗口函数的熟练度

#### 预测问题2：商机表和打标表关联，找出未打标的商机

```sql
SELECT o.opportunity_id, o.brand, o.city
FROM opportunity_table o
LEFT JOIN tagging_table t ON o.opportunity_id = t.opportunity_id
WHERE t.tag_id IS NULL;
```

**讲解要点：**
- `LEFT JOIN + IS NULL` 是找"有A无B"的经典写法
- 可以补充：如果数据量大，也可以用 `NOT EXISTS` 或 `ANTI JOIN`

#### 预测问题3：数据清洗场景 — 重复商机去重

```sql
WITH dedup AS (
    SELECT *,
           ROW_NUMBER() OVER (PARTITION BY brand, city, store_name ORDER BY update_time DESC) AS rn
    FROM raw_opportunity
)
SELECT * FROM dedup WHERE rn = 1;
```

**讲解要点：**
- 窗口函数去重是面试高频题
- 强调 `ORDER BY update_time DESC` 取最新一条

### 方向2：AB实验知识（核心考点 · 必问）

JD明确要求"具备AB实验设计、回收分析等经验"，你简历中也有AB测试经历，这块是加分项。

#### 预测问题1：介绍一下你之前做的AB测试项目

**建议回答（STAR法则）：**

> **Situation**：在长城汽车时，我们需要评估新的门店补贴政策是否有效。
>
> **Task**：设计AB实验，量化新政策对门店转化率的影响。
>
> **Action**：
> 1. **实验设计**：基于用户历史行为分层抽样，确保实验组和对照组在门店规模、历史销量等维度上均衡
> 2. **样本量计算**：根据历史数据估算所需样本量，确保统计功效 ≥ 80%
> 3. **假设检验**：使用双样本t检验分析两组转化率差异，同时计算置信区间
> 4. **AA验证**：实验前做AA测试，验证分流均匀性
>
> **Result**：新政策提升门店转化率约5%，p值 < 0.05，推动业务全国上线。

#### 预测问题2：AB实验中常见的陷阱有哪些？

**回答要点：**
1. **辛普森悖论** — 整体看有效，分群看无效甚至相反 → 需要做分层分析
2. **SRM（样本比不匹配）** — 实验组和对照组人数比例偏离预期 → 需要做SRM检验
3. **组间干扰（Network Effect）** — 实验组和对照组用户互相影响 → 需要做隔离设计
4. **多重比较问题** — 看太多指标容易发现"假显著" → 需要做多重假设检验校正
5. **新奇效应** — 用户对新功能的新鲜感导致短期效果虚高 → 需要延长实验周期

#### 预测问题3：如何确定AB实验需要多少样本量？

**回答：**

> 样本量计算需要4个参数：
> 1. **基线转化率** — 比如当前转化率10%
> 2. **最小可检测效应（MDE）** — 比如想检测出1%的绝对提升
> 3. **显著性水平 α** — 通常取0.05
> 4. **统计功效 1-β** — 通常取0.8
>
> 公式（近似）：
> ```
> n = (Z_α/2 + Z_β)² × [p₁(1-p₁) + p₂(1-p₂)] / (p₂ - p₁)²
> ```
>
> 也可以用Python的 `statsmodels.stats.power.tt_ind_solve_power()` 直接计算。

### 方向3：业务理解与场景题

#### 预测问题1：如果B端商机采集量突然下降50%，你怎么排查？

**建议回答（三步框架）：**

> **第一步：数据验证**
> - 确认数据采集系统是否正常（有无报错、延迟）
> - 确认统计口径是否变更（比如商机定义变了）
> - 确认是整体下降还是某个维度下降
>
> **第二步：维度拆解**
> - 按城市拆：是新开城少了，还是老城采集停了？
> - 按品牌拆：是某个大品牌没采集到，还是普遍下降？
> - 按渠道拆：是线上采集少了，还是线下BD没录入？
>
> **第三步：根因定位**
> - 如果是系统问题 → 联系数仓/采集团队修复
> - 如果是流程问题 → 优化采集SOP
> - 如果是竞争因素（Keeta加入） → 调整采集策略

#### 预测问题2：如何评估商机质量？你会用哪些指标？

**回答：**

> 商机质量可以从以下几个维度评估：
> 1. **转化率** — 商机→签约的比例
> 2. **时效性** — 从采集到分发的平均时长
> 3. **完整度** — 商机信息字段的完整率
> 4. **准确率** — 打标/分类的准确率（可抽样验证）
> 5. **ROI** — 该商机最终带来的GMV / 采集成本

### 方向4：简历深挖

#### 预测问题：你在长城汽车做的销量预测，具体怎么做的？

**建议回答：**

> 我对比了3个模型：
> 1. **ARIMA** — 传统时间序列，适合线性趋势，但对节假日等外部因素捕捉弱
> 2. **Prophet** — Facebook开源，能自动处理节假日效应和趋势变化，可解释性好
> 3. **XGBoost** — 树模型，能加入更多特征（促销、竞品活动等），但需要更多特征工程
>
> **最终方案**：用XGBoost + 时间特征（月份、季度、节假日flag）+ 业务特征（库存量、促销力度），经过网格搜索调参和5折交叉验证，准确率达到85%。

#### 预测问题：你的定价策略论文具体研究了什么？

**建议回答（简洁版）：**

> 我的研究针对网约车平台的三方市场（平台、司机、乘客），核心创新点是：
> 1. **引入乘客有限理性** — 传统模型假设乘客完全理性，我考虑了乘客的信息选择性接受行为，更贴近现实
> 2. **构建差异化定价模型** — 不同乘客群体不同定价，而非统一定价
> 3. **设计PSO-MSA混合算法** — 解决非线性模型的求解难题
>
> 最终平台利润提升约10%，发表2篇SCI论文。
